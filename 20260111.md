# 题目 1：异步限流器 (Asyncio Throttler)
背景： 在工程实践中，我们经常需要调用第三方 API。如果并发过高会被封禁。 要求：

- 实现一个名为 AsyncThrottler 的上下文管理器（Context Manager）。
- 构造函数接受 rate_limit（每秒允许的最大请求数）。
- 确保在多个协程并发调用时，严格遵守 QPS 限制。
- 加分项： 思考如果超过速率，是该抛出异常还是让协程排队等待？


# 题目 2：单例模式的终极实现 (Thread-Safe Singleton Meta)
背景： 在项目工程中，数据库连接池或配置类通常需要单例模式。 要求：

- 编写一个元类 SingletonMeta。
- 确保任何继承自该元类的类都只能创建一个实例。
- 硬核要求： 必须是线程安全的。考虑在多线程环境下，如何避免竞争条件（Race Condition）导致创建出两个实例。


# 题目 3：属性监控器 (Attribute Monitor with Descriptors)
背景： 你需要追踪某些关键变量的变化情况，并记录日志。 要求：

- 利用 Python 的 描述符 (Descriptors) 机制（即 __get__ 和 __set__）。
- 实现一个 ValidatedAttribute 类，它能检查赋值是否符合类型要求，并在每次修改值时打印出旧值和新值。
- 对比： 为什么这里用描述符比直接用 @property 更好？
